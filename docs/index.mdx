 ## Search like a pro

 - Use <strong>filters</strong> in the search box (e.g. <code>flag lang:impl</code>), or just keywords like <em>“release notes prepare”</em>.
 - Prefer <strong>relative links</strong> when you add content (keeps GitHub Pages happy under <code>/prompt-docs/</code>).
 - Add a short <strong>front-matter description</strong> on new pages to improve search snippets.

 ## Contributing

 1. Create a branch, add or edit Markdown under the appropriate section.
 2. Keep file names consistent with the existing pattern (e.g., <code>*.impl.md</code>, <code>*.review.md</code>).
 3. Submit a PR—use the <a href="temp-prompts-organized/20-implementation/review/pr-desc.review.md">PR description helper</a>.
 4. After merge, the site auto-deploys (using <code>mkdocs build</code> + Pages).

 <a href="https://github.com/AcidicSoil/prompt-docs/issues/new">Open a new issue</a> &nbsp; <a href="https://github.com/AcidicSoil/prompt-docs/pulls">Propose a change</a>
